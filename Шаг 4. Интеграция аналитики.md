# Шаг 4. Интеграция аналитики (Яндекс.Метрика)

Документ описывает подключение счётчика Яндекс.Метрики, отслеживание просмотров SPA, JavaScript-целей и e-commerce (`dataLayer`) в проекте StepUp Web MVP.

---

## Цель

- Подключить счётчик Яндекс.Метрики на фронтенде.
- Отслеживать просмотры страниц в SPA (без полной перезагрузки).
- Фиксировать ключевые действия пользователя: вход, регистрация, OAuth, корзина, checkout, покупка.
- Передавать данные e-commerce через `dataLayer` (формат enhanced ecommerce).
- Покрыть модуль аналитики unit-тестами.

---

## Архитектура

```text
index.html (noscript-пиксель)
    ↓
main.js → initMetrika()          — загрузка tag.js, init счётчика, dataLayer
    ↓
router.js → trackPageView()      — hit при каждой смене маршрута SPA
    ↓
страницы → trackGoal / track*()  — цели и e-commerce при действиях пользователя
    ↓
mc.yandex.ru                     — приём hits, reachGoal, ecommerce
```

**Важно:** аналитика работает только на **фронтенде** (`localhost:5173` / Netlify). Backend API (`localhost:3001`) счётчик не загружает.

## Изменения в коде

### Новый модуль: `src/analytics/metrika.js`

Центральный модуль аналитики. Экспортирует:

| Функция / константа   | Назначение                                                                 |
| --------------------- | -------------------------------------------------------------------------- |
| `getMetrikaCounterId()` | Читает `VITE_YM_COUNTER_ID`, fallback `109394182`                       |
| `initMetrika()`       | Загружает `tag.js`, вызывает `ym(id, "init", {...})`, инициализирует `dataLayer` |
| `trackPageView(url)`  | `ym(id, "hit", url)` — просмотр страницы SPA                               |
| `trackGoal(name, params)` | `ym(id, "reachGoal", name, params)` — JavaScript-цель                |
| `trackAddToCart(item)` | `dataLayer` (`ecommerce.add`) + цель `add_to_cart`                        |
| `trackBeginCheckout(cartItems)` | `dataLayer` (`ecommerce.checkout`) + цель `begin_checkout`      |
| `trackPurchase(order, cartItems)` | `dataLayer` (`ecommerce.purchase`) + цель `purchase`          |
| `METRIKA_GOALS`       | Константы имён целей (см. таблицу ниже)                                    |

**Параметры init счётчика:**

```js
{
  ssr: true,
  webvisor: true,
  clickmap: true,
  ecommerce: "dataLayer",
  referrer: document.referrer,
  url: location.href,
  accurateTrackBounce: true,
  trackLinks: true
}
```

**Константы целей (`METRIKA_GOALS`):**

| Константа              | Идентификатор            |
| ---------------------- | ------------------------ |
| `LOGIN_SUCCESS`        | `login_success`          |
| `REGISTER_SUCCESS`     | `register_success`       |
| `OAUTH_YANDEX_SUCCESS` | `oauth_yandex_success`   |
| `ADD_TO_CART`          | `add_to_cart`            |
| `BEGIN_CHECKOUT`       | `begin_checkout`         |
| `PURCHASE`             | `purchase`               |

**Формат e-commerce в `dataLayer`:**

- Валюта: `RUB`.
- Товар маппится в поля: `id`, `name`, `price`, `quantity`, `brand`, `category` (sport), `variant` (size).
- События: `add`, `checkout` (step: 1), `purchase` (с `actionField.id` = номер заказа, `revenue` = сумма).

Все функции трекинга **безопасно завершаются**, если `window.ym` недоступен (AdBlock, скрипт не загрузился).

### Точка входа: `src/main.js`

```js
import { initMetrika } from "./analytics/metrika.js";

initMetrika(); // вызывается до монтирования приложения
```

Счётчик инициализируется один раз при старте SPA.

---

### SPA pageview: `src/router/router.js`

После каждого рендера страницы (включая `popstate`):

```js
trackPageView(`${pathname}${window.location.search}`);
```

Обеспечивает корректные hits при навигации `/` → `/catalog` → `/cart` без перезагрузки.

---

### Fallback без JavaScript: `index.html`

В `<body>` добавлен noscript-пиксель:

```html
<img src="https://mc.yandex.ru/watch/109394182" ... />
```

Фиксирует визит, если у пользователя отключён JavaScript.

### Интеграция на страницах

События отправляются **только при успехе** — не при ошибках валидации или API.

| Файл                             | Что добавлено                                                                 | Условие срабатывания                          |
| -------------------------------- | ----------------------------------------------------------------------------- | --------------------------------------------- |
| `src/pages/LoginPage.js`         | `trackGoal(METRIKA_GOALS.LOGIN_SUCCESS)`                                      | Успешный `loginUser()`                        |
| `src/pages/LoginPage.js`         | `trackGoal(METRIKA_GOALS.REGISTER_SUCCESS)`                                   | Успешный `registerUser()`                     |
| `src/pages/OAuthCallbackPage.js` | `trackGoal(METRIKA_GOALS.OAUTH_YANDEX_SUCCESS)`                               | Успешный `fetchMe(token)` после OAuth         |
| `src/pages/ProductPage.js`       | `trackAddToCart({ id, name, price, sport, brand, size, quantity })`           | Кнопка «Добавить в корзину» после выбора размера |
| `src/pages/CartPage.js`          | `trackBeginCheckout(cart)`                                                    | Кнопка «Перейти к оформлению»                 |
| `src/pages/CheckoutPage.js`      | `trackPurchase(order, cart)`                                                  | Успешный `createOrder()` перед redirect       |

**Не отслеживается:** клик «Войти через Yandex» (только редирект), ошибки login/register, пустая корзина, неудачное создание заказа.

## Цели в интерфейсе Яндекс.Метрики

Создайте в [metrika.yandex.ru](https://metrika.yandex.ru/) → счётчик **109394182** → **Настройки → Цели → JavaScript-событие**:

| Название цели (в UI) | Идентификатор события  | Когда срабатывает в коде              |
| -------------------- | ---------------------- | ------------------------------------- |
| Успешный вход        | `login_success`        | LoginPage, успешный login             |
| Регистрация          | `register_success`     | LoginPage, успешная регистрация       |
| Вход через Yandex    | `oauth_yandex_success` | OAuthCallbackPage, успешный fetchMe   |
| Добавление в корзину | `add_to_cart`          | ProductPage, добавление товара        |
| Начало оформления    | `begin_checkout`       | CartPage, переход к checkout          |
| Покупка              | `purchase`             | CheckoutPage, успешный createOrder    |

**Идентификатор в коде и в интерфейсе Метрики должен совпадать.**

## Частые проблемы

| Симптом                         | Причина / решение                                              |
| ------------------------------- | -------------------------------------------------------------- |
| `typeof ym` → `undefined`       | AdBlock, не тот URL (API `:3001` вместо фронта `:5173`), нет `npm run dev` |
| Нет запросов в Network          | Блокировщик рекламы; отключить для localhost                   |
| Есть init, нет hit при переходах | Проверить навигацию через меню (router вызывает `trackPageView`) |
| Цели не в отчётах               | Не созданы в UI Метрики или идентификатор не совпадает с кодом |
| localhost слабо виден в отчётах | Нормально; смотреть Network, Debugger или «В реальном времени» |
| На Netlify нет Метрики          | Нет `VITE_YM_COUNTER_ID` или не был redeploy после добавления  |
