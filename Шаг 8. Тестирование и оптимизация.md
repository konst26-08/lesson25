## 2. OAuth2 (Yandex)

### 2.1. Что проверяется автоматически

| Файл                                  | Тесты | Сценарий                                                                 |
| ------------------------------------- | ----- | ------------------------------------------------------------------------ |
| `src/server/auth/oauth.test.js`       | 5     | state store, Yandex client URL, email extraction, OAuth service          |
| `src/server/oauth.routes.test.js`     | 4     | redirect на Yandex, callback с token, error=access_denied, invalid state |
| `src/pages/OAuthCallbackPage.test.js` | 2     | цель `oauth_yandex_success` в Метрике после успешного callback           |

```powershell
npm run test:oauth
```

### 2.2. Ручная проверка (local)

1. Заполнить в `.env`: `YANDEX_OAUTH_CLIENT_ID`, `YANDEX_OAUTH_CLIENT_SECRET`, redirect URIs.
2. `npm run api:start` + `npm run dev`.
3. Login → «Войти через Yandex» → callback → редирект на `/account/orders`.
4. DevTools → Network: `GET /api/auth/yandex` → 302 на `oauth.yandex.ru`.

### 2.3. Production (Netlify)

OAuth **не работает** на Netlify без отдельного хостинга API. Фронт на Netlify, API — localhost или VPS.

## 7. Предложения по оптимизации производительности

### 7.1. Frontend (Vite / Netlify)

| #   | Предложение                                                                                          | Эффект                              | Сложность |
| --- | ---------------------------------------------------------------------------------------------------- | ----------------------------------- | --------- |
| 1   | **Lazy loading маршрутов** — `import()` в `routes.js` для Catalog, Checkout, BugAssistant            | Меньший initial JS bundle           | Средняя   |
| 2   | **Preload критичного CSS** — `<link rel="preload">` для main CSS в `index.html`                      | Быстрее First Contentful Paint      | Низкая    |
| 3   | **Cache-Control для assets** в `netlify.toml`: `/assets/*` → `max-age=31536000, immutable`           | Повторные визиты без re-download JS | Низкая    |
| 4   | **Отложенная загрузка Metrika** — `initMetrika()` после `DOMContentLoaded` или `requestIdleCallback` | Быстрее TTI на главной              | Низкая    |
| 5   | **Vite manualChunks** — вынести `bcryptjs`/тяжёлые deps если появятся на фронте                      | Меньше main chunk                   | Средняя   |
| 6   | **Compression** — Netlify gzip/brotli включены по умолчанию; проверить размер `dist/assets/*.js`     | Меньше трафика                      | —         |

**Пример lazy route (рекомендация):**

```js
// routes.js — вместо static import
component: () => import("../pages/CatalogPage.js").then((m) => m.renderCatalogPage);
```

**Пример cache headers (рекомендация для netlify.toml):**

```toml
[[headers]]
  for = "/assets/*"
  [headers.values]
    Cache-Control = "public, max-age=31536000, immutable"
```

### 7.2. Backend (Express / PostgreSQL)

| #   | Предложение                                                                    | Эффект                   |
| --- | ------------------------------------------------------------------------------ | ------------------------ |
| 1   | **Индексы БД** — уже есть на `orders`, `payments`; добавить при росте каталога | Быстрее list/find        |
| 2   | **Pool tuning** — `max`, `idleTimeoutMillis` в `pg.Pool` для production        | Стабильнее под нагрузкой |
| 3   | **ETag / Cache-Control** для `GET /api/products` (public read)                 | Меньше запросов к БД     |
| 4   | **LOG_LEVEL=warn** в production                                                | Меньше I/O на диск       |
| 5   | **Response compression** — `compression` middleware для JSON API               | Меньше payload           |
