# Шаг 3. Интеграция OAuth2 (Yandex ID)

Документ описывает интеграцию **OAuth2 Authorization Code** с провайдером **Yandex ID** в проекте StepUp Web MVP.

---

## Цель

- Вход через Yandex без пароля приложения.
- После OAuth backend выдаёт **существующий JWT** — остальной API (`/api/orders`, `/api/auth/me`) не меняется.
- Email/пароль остаются доступны параллельно.

---

## Архитектура flow

```text
/login → «Войти через Yandex»
    → GET /api/auth/yandex
    → redirect oauth.yandex.ru (state в query)
    → пользователь подтверждает доступ
    → GET /api/auth/yandex/callback?code=...&state=...
    → backend: code → access_token → профиль Yandex
    → find/create user + link user_oauth_accounts
    → jwt.sign(HS256)
    → redirect /login/oauth/callback?token=...
    → фронт: fetchMe(token) → store → /account/orders
```

**CSRF:** параметр `state` генерируется на backend и проверяется однократно (in-memory store, TTL 10 мин).

---

## Настройка Yandex ID (выполнено вами)

| Параметр           | Значение                                         |
| ------------------ | ------------------------------------------------ |
| Приложение         | StepUp Web Dev                                   |
| Redirect URI       | `http://localhost:3001/api/auth/yandex/callback` |
| Права              | email, логин/имя                                 |
| Client ID / Secret | в `.env` (не коммитить)                          |

Переменные окружения:

```env
YANDEX_OAUTH_CLIENT_ID=...
YANDEX_OAUTH_CLIENT_SECRET=...
YANDEX_OAUTH_REDIRECT_URI=http://localhost:3001/api/auth/yandex/callback
OAUTH_FRONTEND_SUCCESS_URL=http://localhost:5173/login/oauth/callback
API_CORS_ORIGINS=http://localhost:5173,http://localhost:4173
```

Шаблон: `.env.example`.

## Backend

### Новые модули

| Файл                                          | Назначение                                 |
| --------------------------------------------- | ------------------------------------------ |
| `src/server/auth/yandexOAuth.js`              | URL authorize, обмен code, профиль Yandex  |
| `src/server/auth/oauthStateStore.js`          | хранение и проверка `state`                |
| `src/server/auth/oauthService.js`             | find/create user, выпуск JWT, redirect URL |
| `src/server/middleware/securityMiddleware.js` | rate limit на auth (в т.ч. OAuth)          |

### API-маршруты

| Метод | Путь                        | Описание                                 |
| ----- | --------------------------- | ---------------------------------------- |
| GET   | `/api/auth/yandex`          | Redirect на Yandex OAuth                 |
| GET   | `/api/auth/yandex/callback` | Callback: code → JWT → redirect на фронт |

Ошибки callback → redirect на фронт с `?error=`:

- `access_denied` — отмена у провайдера
- `invalid_request` — нет code/state
- `oauth_failed` — невалидный state, ошибка token/profile

Маршруты регистрируются только если заданы все OAuth-переменные в `.env`.

### Репозиторий пользователей

Расширены `usersRepository` (Postgres + in-memory):

- `findByOAuth(provider, providerUserId)`
- `linkOAuthAccount(userId, provider, providerUserId)`
- `createFromOAuth({ email, firstName, provider, providerUserId })`

Логика связывания:

1. Есть OAuth-аккаунт → login.
2. Email уже есть (регистрация паролем) → привязка OAuth к user.
3. Иначе → новый user без пароля.

## Безопасность

- Client Secret только на backend (`.env`, не в Git).
- Проверка `state` (anti-CSRF).
- Rate limit на `/api/auth/yandex` и callback.
- JWT: алгоритм `HS256` (как в шаге 2).
- Рекомендуется **перевыпустить Client Secret**, если он попадал на скриншоты.

---

## Netlify / production

Фронт на Netlify не принимает OAuth callback — redirect URI всегда указывает на **API**. Для production нужны:

- публичный HTTPS API;
- redirect URI в Yandex: `https://api-домен/api/auth/yandex/callback`;
- `OAUTH_FRONTEND_SUCCESS_URL=https://....netlify.app/login/oauth/callback`;
- `VITE_API_BASE_URL` при сборке фронта.
